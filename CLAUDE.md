## Architecture

Single-binary Axum app deployed as an AWS Lambda function (`provided.al2023`, arm64). Entry point is `src/main.rs` → `atpr_to::router()`.

**Request flow:**
1. API Gateway HTTP API → Lambda → `router()` in `src/lib.rs`
2. `AppState` (Arc-shared) holds: `OAuthClientType` (jacquard), `reqwest::Client`, `Config`
3. Auth-gated routes use the `AuthSession` extractor (`src/auth.rs`) which reads a signed session cookie (`{did}|{session_id}`)
4. Mutation routes (`/login`, `/shorten`, `/shorten/{code}`) go through `GovernorLayer` rate limiting

**Short URL lifecycle:**
- Create: `POST /shorten` → writes `to.atpr.link` Lexicon record to user's PDS via `com.atproto.repo.putRecord`
- Resolve: `GET /@{handle}/{code}` → tries Slingshot first (2-hop: resolveHandle + getRecord), falls back to direct PDS resolution (3-hop: handle → DID → DID doc → PDS) on non-404 errors
- Delete: `DELETE /shorten/{code}` → deletes PDS record

**Key design notes:**
- `#![warn(missing_docs)]` is enforced — all new public items need `///` docs
- `#[tracing::instrument]` requires `Debug` on args; use `skip_all` to avoid it on request types
- Config loading: compiled defaults → `Config.toml` → `ATPR__` env vars (double-underscore separator via `.prefix_separator("__")`)
- Lexicon types in `src/generated/` are auto-generated by `build.rs` via `jacquard-lexicon` — do not edit manually
- Integration tests in `tests/resolve_integration.rs` use `wiremock` to mock Slingshot; construct `AppState` directly via `router_with_state()`
- On Lambda, rate limiting is per-instance; for global limits, use API Gateway throttling
